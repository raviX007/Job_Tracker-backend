# Email Sending

The API includes a built-in email sender that dispatches cold emails via Gmail SMTP with an optional resume PDF attachment.

---

## How It Works

### Endpoint: `POST /api/emails/{email_id}/send`

1. Fetches the email record from `email_queue`
2. Builds a MIME multipart message (HTML body + plain text alternative)
3. Attaches the resume PDF (if `RESUME_PATH` is set and file exists)
4. Sends via Gmail SMTP (`smtp.gmail.com:587` with STARTTLS)
5. Updates email status to `sent` with `sent_at` timestamp
6. On failure, updates status to `failed` with `last_error`

### Response

**Success:**
```json
{ "status": "sent", "email_id": 1, "to": "founder@startup.com" }
```

**Failure:**
```json
{ "status": "failed", "email_id": 1, "error": "SMTP connection refused" }
```

## Configuration

| Variable | Required | Description |
|----------|----------|-------------|
| `GMAIL_ADDRESS` | Yes | Sender email (your Gmail address) |
| `GMAIL_APP_PASSWORD` | Yes | Gmail App Password (NOT your regular password) |
| `EMAIL_SENDING_ENABLED` | Yes | `true` to enable, `false` to disable |
| `RESUME_PATH` | No | Path to PDF file to attach |
| `TEST_RECIPIENT_OVERRIDE` | No | Redirect ALL emails to this address (testing) |

### Getting a Gmail App Password

1. Go to [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
2. Select "Mail" and your device
3. Google generates a 16-character app password
4. Use this in `GMAIL_APP_PASSWORD` (NOT your regular Gmail password)
5. Requires 2-factor authentication to be enabled on your Google account

| Status | Meaning |
|--------|---------|
| `draft` | Email generated by pipeline, not yet verified |

| `ready` | Email is ready to send |
| `queued` | In the send queue (status set just before SMTP call) |

| `delivered` | Confirmed delivered (future: webhook tracking) |
| `bounced` | Email bounced back |
| `failed` | SMTP error or other send failure |

## Test Mode

When `TEST_RECIPIENT_OVERRIDE` is set, ALL emails are redirected to that address:

```bash
TEST_RECIPIENT_OVERRIDE=your-test@email.com
```

In test mode:
- The subject line is prefixed with `[TEST â†’ original@email.com]`
- The email body is unchanged
- The original recipient is preserved in the subject for debugging
- The `email_queue` record is updated with the test address

This lets you test the full send flow without emailing real recipients.

---

## Resume Attachment

If `RESUME_PATH` is set and the file exists:
- The PDF is attached as `resume.pdf`
- MIME type: `application/pdf`
- Attached to every sent email

If the file doesn't exist, the email sends without attachment (no error).

## SMTP Details

| Setting | Value |
|---------|-------|
| Host | `smtp.gmail.com` |
| Port | `587` |
| Security | STARTTLS |
| Auth | Gmail address + App Password |
| Library | `aiosmtplib` (async) |

## Error Handling

| Error | Behavior |
|-------|----------|
| `EMAIL_SENDING_ENABLED=false` | Returns 400: "Email sending is disabled" |
| Email not found | Returns 404 |
| SMTP connection failure | Status â†’ `failed`, error in `last_error` column |
| Auth failure | Status â†’ `failed`, error logged |
| Attachment missing | Email sends without attachment (no error) |
